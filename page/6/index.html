cocos2d-x ShareSDK

###Cocos2d-x集成ShareSDK出现的问题

异常信息1：

分享完成，回到app过程中异常退出

Fatal signal 11 (SIGSEGV) at xxxxxxxx (code=1), thread xxxx (xxxxxx)
call to OpenGL ES API with no current context (logged once per thread)

Android在回到cocos2d-x时，应该使用OpenGL的Thread：

Cocos2dxGLSurfaceView.getInstance().queueEvent(new Runnable(){do something});

异常信息2：

java.lang.IllegalStateException: Target host must not be null, or set in parameters.
scheme=null, host=null, path=null
版本ShareSDK Android 2.3.7

所有的参数都为null，估计是我们哪里参数转换或传入出了问题，和在线客服的多次沟通下，终于找到问题所在。

在ShareSDKUtils.java文件中找到如下代码
public static void onekeyShare(int platformId, String contentJson)
{
  ....
  HashMap<String, Object> content = hashon.fromJson(contentJson);
  content = nativeMapToJavaMap(content);//注释本行代码
  HashMap<String, Object> map = nativeMapToJavaMap(content);
  ....
}

cocos2d-x集成支付宝问题trade_no

cocos2d-x plugin-x 包含很多第三方的库集成，使用起来非常方便！
官网使用手册 网上也有很多人总结集成的步骤和注意细节点击查看

最近项目需要集成支付宝，按照文档一路下来，除了粗心忘记修改main.cpp的JNI_Onload方法导致报错 javaVM isNULL。main.cpp 做如下修改：

#include "PluginJniHelper.h" 

 jint JNI_OnLoad(JavaVM *vm, void *reserved)
 {
     JniHelper::setJavaVM(vm);
     PluginJniHelper::setJavaVM(vm); 
     return JNI_VERSION_1_4;
 }

大多说产品中需要对支付的交易号进行备份。但在alipay的plugin实现的

 String getOrderInfo(Hashtable info)
方法实现可以看到调用了如下方法获取trade_no。


    /**

 * get the out_trade_no for an order.
 * 获取外部订单号
 * 
 * @return
 */
String getOutTradeNo() {
    SimpleDateFormat format = new SimpleDateFormat("yyyyMMddHHmmss");
    Date date = new Date();
    String strKey = format.format(date);

    java.util.Random r = new java.util.Random();
    strKey = strKey + r.nextInt(10000);
    return strKey;
}


这里的trade_no就是当前时间+一个random值，这不能满足我们的需求，我们的trade_no是自己生成的，需要在交易完成后通过notify_url中传递回到我们的后台处理接口。
我们可以从

 String getOrderInfo(Hashtable info)
方法中根据
String productName = info.get(“productName”);
得出我们也可以将我们的trade_no参数传递的方式传入plugin中来。


MyPurchase::MyPayMode mode = (MyPurchase::MyPayMode) (pMenuItem->getTag() - TAG_PAY_BY_ALIPAY + 1);
pInfo[“productName”] = “100金币”;
pInfo[“productPrice”] = “0.01”;
pInfo[“productDesc”] = “这个是游戏币的测试描述”;
pInfo[“trade_no”] = trade_no;
MyPurchase::sharedPurchase()->payByMode(pInfo, mode);

上面的代码是我们支付的时候调用到的，封装商品信息，在这里加上我们需要的trade_no即可，在getOrderInfo中使用这个值。
至此，重新运行publish.sh即可。

IOS设备抓包方法

在IOS开发中有很多内容和网络通迅相关，这里出现的问题往往不太容易定位，很多时候需要我们还原终端发送或者接收的数据包来对问题进行分析，因此在开发这类App的时候抓包变的很重要。

下面介绍一种在已越狱的ios设备抓包方法：

  1. 安装OpenSSH
  2. 安装tcpdump
  3. 在pc机上通过SecureCRT（也可以使用其他工具），登录ios设备终端，默认root用户密码为alpine（建议修改密码，否则哪天出了个ipad的门，你就火了Y(^_^)Y）
  4. 此时在终端输入tcpdump -s 0 -w capture.pcap命令，把抓取到的包内容保存到capture.pcap文件中
  5. 通过SecureCRT获取设备上的capture.pcap文件到本地，然后使用wireshark进行分析定位问题
